Start a back-and-forth conversation to brainstorm and plan an implementation.

## Your Task
1. **Initialize**: Use `$ARGUMENTS` to understand the goal. 
2. **Explore**: Gather relevant codebase context (existing patterns, constraints).
3. **Multi-Strategy Brainstorming**:
   - Propose **Option A**: Fastest to ship (Minimalist).
   - Propose **Option B**: Most robust/scalable (Gold Standard).
   - Propose **Option C**: Most maintainable (Refactor-heavy).
4. **Iterative Refinement**:
   - This is NOT a one-shot task. 
   - Engage in multiple cycles of "Propose -> Feedback -> Revisit -> QA".
   - Ask clarifying questions to narrow down the best path.
5. **Finalize**: 
   - Once a path is chosen, break it into a task list where each step is small and verifiable.

## Rules
- Always play "Devil's Advocate" to challenge the initial plan.
- Maintain a session state or "Memory Bank" if needed.

## Arguments
`$ARGUMENTS`: The feature or problem you want to brainstorm.
